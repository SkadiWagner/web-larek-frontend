## Базовые классы

### 1. Класс API

Класс __Api__ служит для управления взаимодействием с API посредством HTTP-запросов. Он упрощает выполнение основных операций, таких как отправка запросов и обработка ответов, благодаря обобщенным методам и настройкам.

### Свойства

__baseUrl__: строка, представляющая базовый URL для всех API-запросов. Это начальная точка для формирования URI запросов.
__options__: объект с параметрами, определяющими настройки запроса по умолчанию. Включает заголовки и другие параметры, необходимые для выполнения запросов.

### Методы

### get(uri: string): Promise<any>
Метод отправляет HTTP GET-запрос на указанный uri, который добавляется к baseUrl. Возвращает Promise, который разрешается с данными ответа, обработанными внутренним методом handleResponse.

### Параметры:

__uri:__ относительный путь, добавляемый к baseUrl, чтобы сформировать полный URL для GET-запроса.

### post(uri: string, data: object, method: ApiPostMethods = 'POST'): Promise<any>
Метод отправляет HTTP-запрос с методом POST, PUT или DELETE на указанный uri, включая данные data в теле запроса. Возвращает Promise, который разрешается с данными ответа, обработанными внутренним методом handleResponse.

### Параметры:
__uri__: относительный путь для построения полного URL запроса.
__data__: объект, содержащий данные, отправляемые в теле запроса.
__method__: строка, определяющая HTTP-метод (по умолчанию 'POST').

### protected handleResponse(response: Response): Promise<any>
Защищенный метод, который обрабатывает ответ от API. Возвращает Promise, который разрешается с JSON-данными, если запрос успешен, или отклоняется в случае ошибки.

### Параметры:

__response__: объект Response, полученный от выполнения HTTP-запроса.


### 2. Класс Component<T>

__Component__ — это абстрактный базовый класс, предназначенный для создания пользовательских компонентов, работающих с DOM-элементами. Класс предоставляет методы для управления отображением и состоянием элементов, что упрощает создание интерактивных пользовательских интерфейсов.

### Свойства

__container__: HTMLElement — основной контейнер компонента, в который будут добавляться или обновляться элементы. Этот контейнер передается в конструктор и используется для вставки содержимого компонента.

### Конструктор

constructor(container: HTMLElement)

### Параметры:

__container__: основной DOM-элемент, в который будет встроен компонент.

### Методы

### toggleClass(element: HTMLElement, className: string, force?: boolean): void
Переключает указанный CSS-класс className для элемента element. Если передан параметр force, класс будет принудительно добавлен или удален в зависимости от его значения.

### Параметры:

__element__: DOM-элемент, для которого необходимо переключить класс.
__className__: имя CSS-класса для переключения.
__force__: необязательный логический параметр, который указывает, нужно ли принудительно добавить или удалить класс.

### setText(element: HTMLElement, value: unknown): void
Устанавливает текстовое содержимое для элемента element на значение value.

### Параметры:

__element__: DOM-элемент, в который нужно установить текст.
__value__: новое текстовое значение для элемента.

### setDisabled(element: HTMLElement, state: boolean): void
Меняет состояние блокировки (атрибута disabled) для элемента element на значение state.

### Параметры:
__element__: DOM-элемент, у которого изменяется атрибут disabled.
__state__: логическое значение, указывающее, нужно ли заблокировать (true) или разблокировать (false) элемент.

### setHidden(element: HTMLElement): void
Скрывает элемент element, устанавливая для него стиль display: none.

### Параметры:
__element__: DOM-элемент, который необходимо скрыть.

### setVisible(element: HTMLElement): void
Отображает элемент element, убирая стиль display: none.

### Параметры:

__element__: DOM-элемент, который необходимо отобразить.

### setImage(element: HTMLImageElement, src: string, alt?: string): void

Устанавливает источник изображения src и альтернативный текст alt (если указан) для элемента element.

### Параметры:

__element__: HTMLImageElement, для которого устанавливаются атрибуты.
__src__: URL изображения, устанавливаемого в элемент.
__alt__: необязательный альтернативный текст для изображения.

### render(data?: Partial<T>): HTMLElement
Отрисовывает компонент, используя необязательные данные data, переданные для обновления состояния компонента. Возвращает корневой DOM-элемент (container), в который вставляется содержимое компонента.

### Параметры:

__data__: объект с данными для обновления состояния компонента (необязательный).

### Возвращаемое значение:

__HTMLElement__: корневой DOM-элемент компонента.

### 3.Класс EventEmitter
EventEmitter предоставляет функциональность для управления событиями внутри приложения. Он позволяет добавлять и удалять обработчики, инициировать события и прослушивать все события, обеспечивая гибкий механизм для управления событийной моделью.

### Свойства
___events__: Map<string, Set<Subscriber>> — карта, где ключом является имя события (eventName), а значением — набор подписчиков (callback), которые будут вызваны при возникновении этого события.

### Методы

### on<T extends object>(eventName: string, callback: (event: T) => void): void
Добавляет обработчик callback для события eventName. Обработчик будет вызываться при каждом возникновении этого события.

### Параметры:
__eventName__: имя события, на которое нужно подписаться.
__callback__: функция-обработчик, которая будет вызвана при возникновении события. Получает объект события event типа T.

### off(eventName: string, callback: Subscriber): void
Удаляет обработчик callback для события eventName. Если обработчик был зарегистрирован на это событие, он будет удален из списка подписчиков.

### Параметры:
__eventName__: имя события, для которого нужно снять подписку.
__callback__: функция-обработчик, которую необходимо удалить.

### emit<T extends object>(eventName: string, data?: T): void
Инициирует событие eventName, вызывая все зарегистрированные обработчики с переданными данными data.

### Параметры:
__eventName__: имя события, которое нужно инициировать.
__data__: объект с данными события, который будет передан в обработчики (необязательный).

### onAll(callback: (event: EmitterEvent) => void): void
Добавляет обработчик callback, который будет вызван при возникновении любого события. Обработчик будет получать объект события event, содержащий имя и данные события.

### Параметры:
__callback__: функция-обработчик, вызываемая при любом событии. Получает объект event типа EmitterEvent.

### offAll(): void
Удаляет все зарегистрированные обработчики для всех событий, очищая карту _events.

### trigger<T extends object>(eventName: string, context?: Partial<T>): (data: T) => void
Возвращает функцию-обработчик, которая инициирует событие eventName с переданными данными context. Эта функция может быть использована как коллбек для вызова события в дальнейшем.

### Параметры:
__eventName__: имя события, которое будет инициировано.
__context__: частичный объект данных, который будет объединен с переданными данными при вызове события (необязательный).

### Возвращаемое значение:

(data: T) => void: функция, вызывающая emit для события eventName с объединенными данными data и context.

### 4. Класс Model
Model — это абстрактный класс, предназначенный для создания моделей данных с встроенной поддержкой событий. Он обеспечивает управление данными и оповещение других компонентов о любых изменениях в данных модели, упрощая работу с динамическими данными в приложении.

### Свойства
___data__: Record<string, any> — объект, содержащий данные модели. Поля данных могут быть динамическими и изменяться в зависимости от конкретной реализации модели.
___events__: EventEmitter — экземпляр класса EventEmitter, используемый для управления событиями внутри модели. Он позволяет подписываться на изменения данных и уведомлять других о событиях, происходящих в модели.

### Конструктор
constructor(initialData?: Record<string, any>)

### Параметры:
__initialData__: начальные данные для инициализации модели. Это необязательный объект, который позволяет задать начальное состояние данных модели.

### Методы

### getData<T>(key: string): T
Возвращает значение данных модели по указанному ключу key. Тип возвращаемого значения (T) определяется контекстом использования.

### Параметры:

__key__: строка, представляющая ключ данных, которые нужно получить.

### Возвращаемое значение:
__T__: значение, соответствующее указанному ключу.

### setData(key: string, value: any): void
Устанавливает значение value для данных модели по указанному ключу key. После изменения данных инициирует событие change для оповещения подписчиков.

### Параметры:
__key__: строка, представляющая ключ данных, которые нужно установить.
__value__: новое значение данных для указанного ключа.

### onChange(callback: (data: Record<string, any>) => void): void
Добавляет обработчик callback, который будет вызван при изменении данных модели. Обработчик получает объект с текущими данными модели.

### Параметры:
__callback__: функция-обработчик, вызываемая при изменении данных. Получает объект data с текущими данными модели.

### offChange(callback: (data: Record<string, any>) => void): void
Удаляет ранее добавленный обработчик callback для события изменения данных модели.

### Параметры:
__callback__: функция-обработчик, которую необходимо удалить.

### notifyChange(): void
Явно инициирует событие change, оповещая всех подписчиков о том, что данные модели были изменены. Используется для принудительного обновления данных.

## Слой Presenter 

```TypeScript 
// Класс Events помогает раболтать с такиим событями как: подписка на событие, отправление данных, генерациях новых событий с  контекстом. Имплементирует интерфейс IEvents.
export interface IEvents {
    on<T extends object>(event: EventName, callback: (data: T) => void): void;
    emit<T extends object>(event: string, data?: T): void;
    trigger<T extends object>(event: string, context?: Partial<T>): (data: T) => void;
}

export class EventEmitter implements IEvents {
    _events: Map<EventName, Set<Subscriber>>;

    constructor() {
        this._events = new Map<EventName, Set<Subscriber>>();
    }

    /**
     * Установить обработчик на событие
     */
    on<T extends object>(eventName: EventName, callback: (event: T) => void) {
        if (!this._events.has(eventName)) {
            this._events.set(eventName, new Set<Subscriber>());
        }
        this._events.get(eventName)?.add(callback);
    }

    /**
     * Снять обработчик с события
     */
    off(eventName: EventName, callback: Subscriber) {
        if (this._events.has(eventName)) {
            this._events.get(eventName)!.delete(callback);
            if (this._events.get(eventName)?.size === 0) {
                this._events.delete(eventName);
            }
        }
    }

    /**
     * Инициировать событие с данными
     */
    emit<T extends object>(eventName: string, data?: T) {
        this._events.forEach((subscribers, name) => {
            if (name === '*') subscribers.forEach(callback => callback({
                eventName,
                data
            }));
            if (name instanceof RegExp && name.test(eventName) || name === eventName) {
                subscribers.forEach(callback => callback(data));
            }
        });
    }

    /**
     * Слушать все события
     */
    onAll(callback: (event: EmitterEvent) => void) {
        this.on("*", callback);
    }

    /**
     * Сбросить все обработчики
     */
    offAll() {
        this._events = new Map<string, Set<Subscriber>>();
    }

    /**
     * Сделать коллбек триггер, генерирующий событие при вызове
     */
    trigger<T extends object>(eventName: string, context?: Partial<T>) {
        return (event: object = {}) => {
            this.emit(eventName, {
                ...(event || {}),
                ...(context || {})
            });
        };
    }
}
```

## Архитектура проекта
Проект "Веб-ларек" следует архитектурному паттерну MVP (Model View Presenter), который включает несколько ключевых компонентов для обеспечения модульности, переиспользуемости и легкости сопровождения.
Взаимодействия внутри приложения происходят через события. Модели инициализируют события, слушатели событий в основном коде выполняют передачу данных компонентам отображения, а также вычислениями между этой передачей, и еще они меняют значения в моделях.
 

### Основные части проекта:
### 1. Слой Model:
	Содержит интерфейсы данных для продуктов, заказов и корзины.
* __IProductItem__: Описывает атрибуты продукта.
* __IProductList__: Содержит массив продуктов.
* __IOrder__: Определяет структуру заказа.
* __ICartItem__: Представляет продукт в корзине.
* __ICart__: Управляет содержимым корзины и вычисляет итоговую стоимость.

### 2. Слой View:
	Определяет интерфейсы представлений для отображения данных и взаимодействия с пользователем.
* __IPageView__: Содержит элементы для отображения главной страницы, такие как счетчик корзины и список продуктов.
* __ICardView__: Описывает структуру отдельного продукта на карточке.
* __ICartView__: Представляет содержимое корзины.
* __IFormView__: Отвечает за валидацию форм.
* __IModalView__: Управляет модальными окнами.

### 3. Слой Presenter:
	Обеспечивает управление бизнес-логикой и взаимодействием между моделью, представлениями и API.
* __IEvents__ предоставляет механизмы для обработки событий.
* __IApiClient__ определяет интерфейс для работы с внешним API, таким как получение продуктов и создание заказов.

### Взаимодействие между частями проекта:
### Модульность:

* Компоненты представления используются для отображения данных и обработки пользовательского ввода.
* Презентеры и контроллеры управляют бизнес-логикой и взаимодействием с моделью данных и API.

### Обмен данными:

* Данные о продуктах и заказах передаются между моделью, представлениями и контроллерами через определенные интерфейсы и методы.

### Пример процесса в приложении:
* Взаимодействие с пользователем осуществляется через представления, которые обновляются при изменении данных в модели.
* Событийная модель (IEvents) используется для управления событиями и уведомлений между компонентами.

### Основные типы и интерфейсы:

* __IProductItem__, __IProductList__, __IOrder__, __ICartItem__, __ICart__: модели данных для продуктов, заказов и корзины.
* __IPageView__, __ICardView__, __ICartView__, __IFormView__, __IModalView__: __интерфейсы__ представлений для отображения информации и взаимодействия с пользователем.
* __IEvents__, __IApiClient__: интерфейсы для управления событиями и взаимодействия с внешним API.
